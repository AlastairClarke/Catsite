Linux Part 1: Functions

tar

The tar function allows you to zip and unzip (extract) files. Zipped files take up less storage and are easier to transport.
$ man tar tar [-]xvzf [pathname ...] tar -cf archive.tar foo bar tar -tvf archive.tar tar -xf archive.tar Suffixes after .tar: .gz, .bz, .bz2, .xz, .Z, .zip, .7z, .rar.
What is the difference between tar.gz, tar.xz, bz?
Seems that there is no difference between the name 'foo.tar' and 'foo.tar.gz'. However the .gz suffix lets the other know that the files were compressed using gzip.
Does it matter? Do I have to know how the files were compressed in order to extract them? You can gzip a folder using tar czvf archive.tar foo*.jpg and unzip using tar xvf archive.tar .

ln

Allows you to make softlinks to files such as binaries. For example, to run the program foo, I have to cd to where foo is and then run ./start-foo. Instead I could create a softlink to start-foo in my /usr/local/bin/ folder. But I want to go further, start-foo is too long a name. I just want to call it foo, so that all I have to do is type: foo, anywhere and anytime and the program runs. I could change the name of the binary.
A basic softlink to start-foo. sudo ln -s ~/.../start-foo /usr/local/bin/start-foo

file

You can use file to learn more about a particular file.

man

The manual's section on man provides a description of the synopsis. Bold text should be typed, italic text should be replaced with an argument, any or all arguments within [] are optional. They are called options. man -k printf # Searches for keyword printf

  • To view contents of file: cat, more (use spacebar to scroll)
  • Count words, lines in a file: wc
  • Output to new file ">", append ">>"
  • Pipe output into second command using "|"
  • whereis bash
  • whoami (hopefully not root)
  • To find where the program runs from: which -a ls
  • More information: man hier, info info, man proc
  • Check spelling: grep occasion /usr/share/dict/words
The commands man and info should be your first port of call when you get stuck. Experienced users read the documentation every day: beginners should read it all the time.

Linux Part 2: bash

Getting started

"The shell is an advanced way of communicating with the system." Intro to Linux
bash stands for Bourne Again SHell. It is the standard GNU shell.
  1. Create a ".sh" file - shell script.
  2. First line: #!/bin/bash
  3. Run script ./myscript.sh. If permission denied: chmod +x myscript.sh.
Note that "." means the current directory cf. ".." the parent directory.
To only grant the user the permission to execute a file: chmod u+x myscript.sh.
The test function should be very useful. Allows you to test whether file1 is newer or older than file2, whether file1 and file2 exist.

The code echo $PATH displays the content ($) of the variable PATH.

What was that command I used?

If you need to recall a command you used, the following is useful. For example, I use openconnect occasionally so I forget the order of the command. Also play with Ctrl+R.
$ grep -a open .bash_history
